# Constraints

## Business Constraints

- Initial target: mid-market B2B SaaS teams with 5 to 25 PMs.
- Preferred wedge: Notion plus Linear teams with existing evidence chaos.
- SMB/founder segment is a possible low-touch future path but is not the primary paid wedge.
- Enterprise can be considered only if admin controls and security evidence are addressed.

## Product Constraints

- External tool actions must be dry-run first.
- No live Notion, Linear, GitHub, npm, or network writes are allowed in this test pack.
- AtlasBoard beta does not support GitHub Issues preview yet.
- External writes are disabled in the synthetic analytics.
- Raw interview recordings are not supported in this test; only text snippets and summaries are used.

## Evidence Constraints

- Pricing willingness is directional, not validated.
- No procurement timeline is confirmed for enterprise buyers.
- No usability testing has been run on the proposed design brief.
- No paid conversion data exists.
- Data quality for support imports is inconsistent.
