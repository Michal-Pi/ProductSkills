# Expected Observations: pm-docs

## Required Sections

- PRD quality review.
- Evidence-backed revised PRD or blocked PRD sections.
- Assumptions, non-goals, success metrics, risks, open questions, and next actions.
- Unsafe tooling language corrections.

## Evidence It Should Cite

- Rough PRD gaps.
- INT-001, INT-002, INT-003, SUP-001, SUP-003, SUP-008, SUP-009.
- Usage analytics by segment.
- Constraints on dry-run external actions.

## Risks To Flag

- Rough PRD says "send to Notion" without dry-run safety.
- Metrics are vague.
- Target customer is too broad.

## Must Not Invent

- Approval status for the rough PRD, validation completion, live sync capability, or detailed technical architecture.

## Should Block Or Ask Questions When

- Evidence is insufficient for scope, metrics, security claims, or live external writes.
