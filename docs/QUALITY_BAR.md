# Quality Bar

The package should not optimize for "more PM templates." It should optimize for decision quality, evidence quality, and safe tool handoff.

## Product Artifact Standard

Every major artifact should include:

- Clear objective.
- Target customer/user.
- Evidence and source notes.
- Assumptions.
- Non-goals.
- Success metrics.
- Risks.
- Open questions.
- Recommended next actions.

## Evidence Standard

Strong outputs distinguish:

- Direct customer evidence.
- Quantitative product evidence.
- Market/competitive evidence.
- Internal stakeholder opinion.
- Model inference.
- Missing evidence.

## Prioritization Standard

The prioritization skill must not default blindly to RICE. It must choose the method:

- RICE for reach/impact/confidence/effort comparisons.
- WSJF or cost of delay for time-sensitive sequencing.
- ICE for lightweight experiment backlogs.
- MoSCoW for release-scope negotiation.
- Opportunity scoring for customer-discovered opportunities.
- Weighted scorecards for strategy-specific tradeoffs.

## Growth Standard

Growth outputs must identify:

- Growth model.
- Loop or funnel shape.
- Bottleneck.
- Target metric.
- Segment.
- Experiment hypothesis.
- Guardrail metric.
- Decision threshold.

## Tooling Standard

Notion/Linear operations must:

- Discover workspace IDs before writing.
- Preview payloads.
- Ask for confirmation.
- Write idempotently using external ID maps.
- Save checkpoints for batch operations.
- Never claim true rollback when only manual revert is possible.
