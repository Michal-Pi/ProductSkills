# One-Pager

> Output template for `pm-docs/one-pager`. Narrative-first (prose, not checklists). Fits ≤500 words. Replace placeholder text in italics.

## Header

- **Title:** _the bet in 4-8 words_
- **Author:** _PM_
- **Date:** _YYYY-MM-DD_
- **Reader / audience:** _name(s) or role(s)_
- **Ask:** _approval | written feedback | FYI no action_
- **Status:** _draft | in review | decided_

## The bet

_One sentence. What we would do, and for whom. State it before any context._

## Why now

_2-3 sentences. The trigger, the window, why this beats waiting. If no time-bound trigger exists, say so explicitly — opportunistic bets are valid but must be named as such._

## Customer and problem

_2-3 sentences. Who is affected, what hurts them today. Cite a discovery artifact, interview corpus, or quantified opportunity. Inline placeholder evidence is allowed only when explicitly flagged "uncited — to confirm."_

- **Evidence:** _<EVID-ids or source links>, grade H / M / L_

## Proposed solution outline

_2-3 sentences at concept level — what the customer would experience, not how engineering would build it. If solution detail belongs here, the artifact is a PRD, not a one-pager._

## Expected outcome

_The target metric movement with baseline + target + window. Cite `pm-metrics` if available; if the metric is unknown, flag the gap and route to pm-metrics rather than fabricate a number._

- **Metric:** _name (cited)_
- **Baseline:** _current value_
- **Target:** _expected value_
- **Window:** _by when_

## Key risks

_Top 2-3 risks, steelmanned. One sentence each. No strawmen._

- _Risk 1_
- _Risk 2_
- _Risk 3 (optional)_

## Ask

_The single explicit ask, as an imperative sentence. "We are asking for approval to staff this in Q3." "We are asking for written feedback by <date>." "This is an FYI; no action needed."_

## Open questions

- _Question 1 — owner if known_
- _Question 2 — owner if known_

## Sources

- _Linked discovery / strategy / metric artifacts behind the claims above_
