# PM GTM

Prepare launches and post-launch learning loops without separating product intent from market execution.

## Core Procedures

- Use `procedures/launch-readiness.md` for launch decisions.
- Use `procedures/positioning-messaging.md` for audience-specific messaging.
- Use `procedures/release-notes.md` for customer-facing or internal release notes.
- Use `procedures/post-launch-review.md` after launch.

## References

- Use `../../references/methods/launch-planning.md` for launch type and readiness checks.
- Use `../../references/checklists/launch-method-coverage.md` before finalizing GTM or launch artifacts.
- Use `../../templates/launch-plan.md` for launch plans.

## Output Standard

GTM artifacts should include audience, message, channel, owner, timing, risk, customer impact, internal enablement needs, and success metrics.
