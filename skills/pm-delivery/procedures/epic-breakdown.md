# Epic Breakdown

Use when turning product scope into epics.

## Steps

1. Restate product outcome and target user.
2. Identify major user workflows or capabilities.
3. Split epics by user value, risk, or release boundary.
4. Preserve assumptions, non-goals, and dependencies.
5. Add measurable completion criteria.
6. Identify sequencing constraints.

## Output

Use `../../../templates/epic.md` for each epic or a compact epic map.

## Done when

- At least two epics are produced, each with a measurable completion criterion and a named target user.
- Each epic carries explicit assumptions, non-goals, and dependencies; items inherited from the source scope are cited back to that source.
- Splits are justified by user value, risk, or release boundary (not by system layer), and sequencing constraints between epics are stated.
- When product outcome or target user is missing from the input, the procedure stops and asks rather than inventing scope to fit a template.
