# Prototype Spec

Use when specifying a prototype for validation or stakeholder review.

## Steps

1. Define what decision the prototype should answer.
2. Choose fidelity based on risk: sketch, clickable, coded, data-backed, or concierge.
3. Define paths, states, inputs, outputs, and realistic content.
4. Identify what should not be built.
5. Define usability tasks and success signals.
6. Route to `pm-validation` when the prototype is ready to test.

## Output

Return prototype scope, screens or flows, states, data needs, validation tasks, and risks.
