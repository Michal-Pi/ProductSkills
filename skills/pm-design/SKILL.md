# PM Design

Create design artifacts that are specific enough for designers and engineers, but still grounded in user outcomes and evidence.

## Core Procedures

- Use `procedures/wireframe-brief.md` for early flow and screen briefs.
- Use `procedures/prototype-spec.md` when a prototype should answer a product decision.
- Use `procedures/usability-test-plan.md` for task-based usability tests.
- Use `procedures/design-review.md` for design readiness review.

## Output Standard

Design artifacts must include target user, user goal, scenario, scope, states, edge cases, content needs, success criteria, and open questions.
