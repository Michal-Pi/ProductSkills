# Spec Review

Use when reviewing a PRD, product spec, or technical brief for quality.

## Steps

1. Check objective, customer, evidence, scope, non-goals, metrics, risks, and open questions.
2. Identify unsupported claims and fake certainty.
3. Check if requirements are testable and delivery-ready.
4. Separate blocking issues from improvements.
5. Recommend revise, validate, align, or proceed.

## Output

Return findings by severity, missing sections, risky assumptions, and next action.
