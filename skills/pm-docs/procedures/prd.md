# PRD

Use when drafting or revising a product requirements document.

## Steps

1. Clarify objective, target user, and decision status.
2. Pull evidence from discovery, analytics, market, or stakeholder sources.
3. Separate assumptions and open questions from confirmed facts.
4. Define scope, non-goals, solution outline, metrics, risks, and rollout notes.
5. Add acceptance-level product requirements without over-specifying implementation.
6. Flag gaps that block delivery.
