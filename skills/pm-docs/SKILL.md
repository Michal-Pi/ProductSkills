# PM Docs

Create evidence-backed product artifacts that can survive review by product, design, engineering, GTM, and leadership.

## Core Procedures

- Use `procedures/prd.md` for product requirements documents.
- Use `procedures/decision-log.md` for durable product decisions.
- Use `procedures/spec-review.md` for PRD, spec, or brief review.

## Templates

- Use `../../templates/prd.md` for PRDs.

## References

- Use `../../references/methods/evidence-grading.md` when reviewing evidence strength.
- Use `../../references/checklists/docs-quality.md` before finalizing PRDs or specs.
- Use `../../schemas/skill-output-envelope.schema.json` as the shared output contract for major artifacts.

## Output Standard

Major docs should include objective, customer, evidence, assumptions, scope, non-goals, solution outline, success metrics, risks, open questions, and next actions.
