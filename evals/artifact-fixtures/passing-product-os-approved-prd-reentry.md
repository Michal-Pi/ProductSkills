# entry_classification

Lifecycle status is approved_for_delivery because the PRD includes scope and non-goals, assumptions, risks, and success metrics.

# validation_not_required

Validation not required because the artifact is already approved for delivery. The workflow records earlier evidence gaps as risk but does not force validation replay.

# delivery_readiness_gate

Readiness check confirms scope and non-goals, open questions, assumptions, dependencies, and delivery decision.

# delivery_split

Includes epics or stories, acceptance criteria, edge cases, failure states, dependencies, and sequencing constraints.

# tool_dry_run_preview

Dry-run payload is shown before write. Explicit confirmation is required for target team, idempotency keys, issue list, and dry_run_payload_hash. External writes are disabled during evals, and rollback is not overstated.

# launch_readiness

Launch readiness covers audience, customer impact, support readiness, enablement readiness, rollout risk, success metrics, and post-launch learning plan.

# handoff

Handoff target is workflow-prd-to-linear-delivery, then pm-gtm. Approval gates cover ambiguous scope, tool writes, and launch risk.
