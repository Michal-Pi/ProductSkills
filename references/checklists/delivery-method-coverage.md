# Delivery Method Coverage Checklist

Use this checklist when reviewing delivery handoffs.

- Product outcome and target user are preserved.
- Scope, non-goals, assumptions, and open questions are included.
- Epics are split by user value, workflow, risk, or release boundary.
- Stories are valuable, testable, and small enough to plan.
- Acceptance criteria include primary path, edge cases, and failure states.
- Dependencies and sequencing constraints are visible.
- Tool payloads are dry-run previews until explicit user confirmation.
